# Accessibility rules (UI changes)

Apply to every change that touches markup, styles, or interaction:

1. Semantics first: real <button>/<a>/<label>/<nav>/headings - div+onClick
   needs a written reason plus role and tabindex.
2. Keyboard: everything clickable is reachable and operable by keyboard;
   visible focus style; no focus traps; Escape closes overlays.
3. Labels: every input has a label; every icon-only control has an
   aria-label; images have alt (empty alt for decorative).
4. Contrast: text >= 4.5:1 (3:1 for large text); never color as the only
   signal.
5. Motion and time: respect prefers-reduced-motion; no content that flashes
   or auto-disappears before it can be read.
6. State changes (errors, toasts, loading) are announced - aria-live for
   dynamic messages.
